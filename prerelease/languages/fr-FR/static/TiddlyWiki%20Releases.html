Versions de TiddlyWiki

1 juin 2015 à 9h46

Voici la description des récentes mise à jour de TiddlyWiki5. Voir TiddlyWiki5 Versioning pour plus de détails sur la nomination des versions.

(Les versions Bêta et Alpha sont listées séparément).

Release 5.1.23

Released 24 décembre 2020 at 13:29

See GitHub for detailed change history of this release

Performance Improvements

  • filter execution to use a more efficient linked list structure for intermediate results
  • widget mechanism to cache the result of parsing macros which don't use text substitution
  • page templates to use a single VarsWidget instead of several SetVariableWidgets, for improved performance and easier debugging
  • ListWidget to not initialize parsers for blank emptyMessage attributes

Usability Improvements

  • "references" to "backlinks" in the tiddler info panel
  • warning message about using the online plugin library with the client-server configuration
  • favicon for the prerelease edition
  • normalize.css's styling of search input fields and to a modern fork of normalize.css
  • unneeded editor toolbar buttons when editing SVG tiddlers
  • global keyboard shortcut for switching layouts
  • the .tid exporter when more than one tiddler to export

Import Mechanism Improvements

  • support for renaming tiddlers and a warning about overwriting existing tiddlers
  • colour coding for import items that will overwrite existing tiddlers, and for other warnings
  • other warnings in the import listing
  • content type of imported .tid files that do not have a type field
  • extraneous system tiddlers created during import process

Palette Improvements

  • the $:/PaletteManager to show "indirect" colours (ie, colours defined by another <<colour>> macro)
  • new "Desert Sand" palette
  • new "Cupertino Dark" palette
  • "Solarized Dark" palette
  • (and ) the Vanilla theme to optionally use palette colours for the browser selection outline
  • "Gruvbox" palette readability of toolbar buttons

Keyboard Handling Improvements

  • support for navigating the search and new tag dropdowns via the keyboard. The keyboard-driven-input Macro can be used to add this capability to other dropdowns
  • a tabIndex property to the tiddler editor input area to make it easier to use the tab key to move between edit controls
  • keyboard support for cycling through the tabs in $:/AdvancedSearch
  • keyboard support for navigating the field name dropdown in the Edit Template
  • keyboard support or navigating the type field input in the Edit Template
  • keyboard support for using the insert wikilink toolbar dropdown in the Edit Template
  • a keyboard shortcut for saving the wiki
  • a keyboard shortcut for deleting a field in the edit template
  • a keyboard shortcut to change the sidebar layout

Widget Improvements

  • EventCatcherWidget for low level event handling
  • new LogWidget and ActionLogWidget to help debugging
  • new ActionConfirmWidget to prompt the user for simple yes/no questions
  • ButtonWidget and DroppableWidget so that changing the class attribute does not trigger a refresh. This makes it easier to use classes to trigger CSS animations
  • the RevealWidget to optionally dynamically refresh popup positions when the state tiddler changes
  • support for disabled attribute to EditWidget, EditTextWidget, CheckboxWidget, RadioWidget and RangeWidget
  • support for disabled attribute to ButtonWidget
  • the MacroCallWidget to be able to optionally render the raw text of the macro (previously the output was always wikified)
  • the LinkWidget to work within SVG elements
  • accept attribute to the BrowseWidget
  • the ActionPopupWidget to allow popups to be dismissed
  • the EditWidget to pass all attributes through to the sub-widget
  • the LinkCatcherWidget to pass keyboard modifier status to actions
  • the RadioWidget to support actions
  • the RangeWidget to support actions
  • autocomplete attribute to the EditTextWidget and EditWidget
  • incorrect behaviour of default values with lookup Operator
  • has Operator when used with the tags field
  • crash when using edition Operator in the browser
  • class attribute of DroppableWidget

Filter Improvements

New capabilities:

  • support for multiple operands for filter operators

New filter run prefixes:

  • support for named filter run prefixes
  • new :filter filter run prefix which is analagous to the new filter Operator but applies to a filter run
  • new :intersection filter run prefix to get the intersection of two filter runs
  • new :reduce filter run prefix

New operators:

  • new filter and reduce operators for processing lists of items
  • new power Operator and log Operator
  • new draft Operator to check if a tiddler is a draft of another tiddler
  • new search-replace Operator to search and replace in strings
  • new enlist-input Operator to parse its input titles as a title lists
  • new slugify Operator and duplicateslugs Operator for generating human readable filenames/URLs
  • new sortsub Operator for sorting by a user defined subfilter
  • new format Operator for formatting dates
  • new reduce Operator to apply a subfilter to each input title in turn, accumulating a single value
  • new filter Operator to apply a subfilter to each input title and return the titles that return a non-empty result from the subfilter
  • new toggle Operator to toggle a title in a list
  • new cycle Operator
  • new pad Operator

Other improvements:

  • the trim Operator to optionally trim a given string instead of whitespace, and trim from front, back, or both sides of input tiddlers
  • jsonstringify Operator and stringify Operator to support "rawunicode" suffix

Tiddler File Naming Improvements

the mechanism for Customising Tiddler File Naming to:

  • Save drafts to a special folder for each user
  • Allow filters in $:/config/FileSystemPaths to change the path of a tiddler file on disk
  • Allow filters in $:/config/FileSystemExtensions to change the extension of a tiddler file on disk
  • th-make-tiddler-path hook for low-level control of tiddler path construction
  • tiddlywiki.files Files specification with isEditableFile attribute allowing files to be saved back to their original location

Hackability Improvements

  • basic support for switching page templates
  • the WidgetMessage: tm-scroll message to allow the target element to be specified by a CSS selector, making it possible to scroll to positions within a tiddler
  • modals to incorporate a NavigatorWidget so that links work as expected
  • the Table-of-Contents Macros to support custom link targets
  • new Hyperdrive saver for use with Beaker Browser v1.0
  • favicon mechanism to support _canonical_uri images
  • support for recording the startup timestamp in $:/info/startup-timestamp (see InfoMechanism)
  • support for SVG favicons
  • th-saving-tiddler hook to include information about the draft tiddler (see https://tiddlywiki.com/dev/#Hook%3A%20th-saving-tiddler)
  • WidgetMessage: tm-rename-tiddler to control whether relinking occurs
  • WidgetMessage: tm-import-tiddlers to override the title $:/Import and to better control whether the import tiddler is opened automatically
  • the internal <$element> widget to add a hook so that plugins can intercept DOM node creation
  • original event to tm-navigate event
  • the password prompt to enable it to be customised
  • syncer to enable syncadaptors to customise the login prompt
  • the tabs Macro to support actions and explicitState attributes
  • (and ) filters used for syncing on node.js and saving the single file version to exclude multiple story lists and history lists based on their prefix, as well as multiple tiddlers that might be used for the import process
  • post-render startup actions
  • support for username/password parameters for tm-login message
  • support for the content type image/jpg (the correct type is image/jpeg but the misspelling is common so most browsers now support it)
  • support for an override saver
  • utility CSS classes to replace use of &nbsp; to introduce visual separation
  • option to configure the tag used for TableOfContents in the menubar
  • the KeyboardWidget to not trap keys if there are no actions to be invoked
  • buttons to the Edit Template toolbar for the editor-height and the stamp tool for tiddlers of type application/javascript,application/json and application/x-tiddler-dictionary
  • all instance of the tabs Macro in the core to use the explicitState attribute
  • support for the meta key as a modifier in actions
  • support for $:/info/darkmode to the InfoMechanism, reflecting the browser dark mode vs. light mode setting
  • story.js to remove dependency on wiki.js for story start up and navigator

Plugin Improvements

JSZip Plugin

the JSZip Plugin with the ability to dynamically create Zip files, and thus to conveniently build multi-file static sites within the browser

Consent Banner Plugin

the new Consent Banner Plugin plugin helps make websites that are compliant with "cookie legislation" such as the EU General Data Protection Regulation. It presents a banner inviting the user to accept or reject cookies, keeping track of their consent in local storage so that the banner can be hidden on subsequent visits.

By default, content embedded with <iframe>, <embed> and <object> is blocked unless the user consents to accept cookies.

Consent status is available via a configuration tiddler so that it is possible to construct content that behaves differently depending upon whether consent has been granted. As an example, a macro is provided for embedding YouTube videos that automatically uses the youtube-nocookie.com variant of video URLs unless the user has accepted cookies.

Please note that using this plugin does not guarantee compliance with any particular legislation. You will need to understand the technical issues specific to your situation, and if necessary seek legal advice.
